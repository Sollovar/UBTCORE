---
name: Port 8080 conflict between api-server and Go Backend
description: artifacts/api-server listens on port 8080 and conflicts with the Go Backend
---

The `artifacts/api-server: API Server` workflow (Express) uses port 8080, which is the same port as the Go Backend. When both are running, the Go Backend fails to bind.

**Why:** The artifacts/api-server is a pre-existing Express API skeleton; the Go Backend is the real matching engine that should own port 8080.

**How to apply:** The artifacts/api-server workflow auto-stops and is not included in the Go Backend startup. Do not restart it. The Go Backend workflow is the authoritative service on port 8080.
